## 8. Async/Await와 Promise의 차이 ✅

둘다 자바스크립트에서 비동기 처리를 다룰 수 있는 방법이다. Promise의 then, Async/Await는 용도는 같지만, 간결성, 에러 핸들링, 에러 위치 확인 측면에서 차이가 있다. 이외에도 Async/Await은 디버그를 할 때 then과 달리 정확한 위치를 지정할 수 있는 장점이 있다.

## 23. 동기와 비동기의 차이

- 동기는 요청과 그 결과가 동시에 일어난다는 뜻이다. 요청을 하면 시간이 얼마나 걸리던지 요청한   자리에서 결과가 주어져야 한다.
  순서에 맞춰 진행되는 장점이 있지만, 여러 가지 요청을 동시에 처리할 수 없다.
- 비동기는 요청과 결과가 동시에 일어나지 않을 거라는 뜻이다.
  하나의 요청에 따른 응답을 즉시 처리하지 않아도, 그 대기 시간동안 또 다른 요청에 대해 처리 가능한 방식이다.
  여러 개의 요청을 동시에 처리할 수 있는 장점이 있지만 동기 방식보다 성능이 떨어질 수도 있다.

## 24. Array vs LinkedList
Array는 index를 통해 빠르게 접근할 수 있고 삽입이나 삭제 시 뒤쪽 모든 요소를 이동시켜야 하므로 비효율적이다. LinkedList의 각 요소는 자신의 다음 요소를 가리키는 포인터를 가집니다. 구조 재구성이 필요없어 삽입, 삭제가 빠르나 접근 속도가 느리다.

## 27. 아토믹 디자인 패턴에 대해 아는가?
 아토믹 디자인 패턴은 웹 페이지를 유저 인터페이스의 기본 단위인 원자(Atoms)부터 조합체(Molecules)까지의 순차적인 과정을 거쳐 전체 페이지의 디자인을 구성하는 것을 목적으로 한다. 
원자(Atoms): 가장 작은 단위의 UI 요소 (ex. 버튼, 텍스트, 입력상자 등)
분자(Molecules): 원자들의 조합으로 구성된 더 큰 단위의 UI 요소
생화학 물질(Organisms): 분자들의 조합으로 구성된 복잡한 UI 요소
템플릿(Templates): 생화학 물질의 조합으로 구성된 페이지의 구조, 실제 콘텐츠가 없는 page 수준의 스켈레톤이라고 생각하면 된다.
페이지(Pages): 템플릿을 기반으로 실제 콘텐츠를 담고있는 결과물

업데이트 : 컴포넌트에서 props가 바뀔 때, state가 바뀔 때, 부모 컴포넌트 리렌더링 때, this.forceUpdate로 강제 렌더링할 때 업데이트
언마운트 : DOM에서 컴포넌트를 제거하는 과정

## 39. react 클래스형과 함수형의 차이
 클래스형과 함수형은 하는 일은 동일하지만 코드 구조에서 약간 차이가 있고, state 관리 기능 등을 코드에 구현을 하고 함수형은 hook을 통해 구현한다. 현재 리액트 공식문서에서는 함수형 컴포넌트와 훅을 이용하는 것을 권장함

## 40. 타입스크립트란?
 자바스크립트를 기반으로 정적 타입 문법을 추가한 컴파일 언어이며 타입스크립트 컴파일러 또는 바벨을 통해 자바스크립트 코드로 변환됩니다. 코드에 목적을 명시하고 목적에 맞지 않는 타입의 변수나 함수들에서 에러를 발생시켜 버그를 사전에 제거합니다.

## 41. Angular와 React의 차이점
앵귤러는 프레임워크이고 리액트는 라이브러리이다. 앵귤러는 양방향 바인딩을 지원하고 리액트는 단방향 바인딩을 지원한다. 양방향 바인딩은 작성해야할 코드량이 줄어드는 장점이 있지만, 변화에 따라 DOM 전체를 랜더링 해주므로 성능이 감소되는 경우가 있다. 단방향 바인딩은 데이터 흐름이 단방향이어서 코드를 이해하기 쉽고 데이터의 추적과 디버깅이 쉽다. 하지만 변화를 감지하고 화면을 업데이트하는 코드를 매번 작성해야 된다는 단점이 있다.

## 42. SEO란?
검색 엔진 최적화를 가리키는 용어로, 
웹페이지 검색엔진이 자료를 수집하고 순위를 매기는 방식에 맞게 페이지를 구성하여  웹사이트가 검색 결과에 더 잘 보이도록 최적화하는 과정

## 43. 이벤트 위임이란?
 하위 요소에 각각 이벤트를 붙이지 않고 상위 요소에서 하위 요소의 이벤트들을 제어하는 방식

## 44. 메모이제이션이란?
 동일한 계산을 반복해야 할 때, 이전에 계산한 값을 메모리에 저장함으로써 동일한 계산의 반복 수행을 제거하여 프로그램 실행 속도를 빠르게 하는 기술

## 45. 이벤트 버블링이란
 특정 화면 요소에서 이벤트가 발생했을 때 해당 이벤트가 상위의 화면 요소들로 전달되어 가는 특성을 의미한다.

## 46. 부모에서 자식으로 이벤트상속방법
 이벤트 캡처링으로 부모에서 자식으로 이벤트를 상속할 수 있습니다. 이벤트 캡처링은 이벤트 버블링과 반대로 상위 요소에서 하위 요소로 탐색하며 이벤트를 전파
 하는 방식입니다.

## 47. 이벤트버블링을 막는 방법
 event.stopPropagation()를 사용하면 클릭한 요소의 이벤트만 발생시키고 상위 요소로 이벤트를 전달하는 것을 방해합니다

## 48. 이벤트버블링 활용 방법
 하위 요소에 각각 이벤트를 붙이지 않고 상위 요소에서 하위 요소의 이벤트들을 제어하는 방식으로 이벤트 버블링을 활용할 수 있습니다.이벤트 버블링을 응용하여 부모 엘리먼트에 리스너를 위임하여 부착하면, 리스너의 개수를 1개로 줄여 최적화할 수 있습니다.
## 49. React, 왜 사용하시나요?
 Virtual DOM을 통해 최적화된 랜더링을 할 수 있어 컴포넌트 단위의 개발이 가능하게 해준다.  컴포넌트를 사용한 재사용이 가능하고 유지보수에 용이하다.
